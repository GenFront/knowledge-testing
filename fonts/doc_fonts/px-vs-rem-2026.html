Конвертер PX → REM (база 18px)

Быстрая конвертация

Формула: rem = px / 18

1.333rem
/* Автоматический расчёт через calc() */
.element {
padding: calc(1rem * 24 / 18); /* 24px в rem */
margin: calc(1rem * 36 / 18); /* 36px в rem */
}

Частые значения (шпаргалка)

  • 4px = 0.222rem
  • 8px = 0.444rem
  • 12px = 0.667rem
  • 16px = 0.889rem
  • 24px = 1.333rem
  • 32px = 1.778rem
  • 48px = 2.667rem
  • 64px = 3.556rem

💡 В 2026 многие дизайнеры сразу дают макеты в REM!

Адаптивность с REM (ключевой нюанс 2026)

:root {
/* 2026: умная адаптивная база */
--base-size: clamp(16px, 1vw + 14px, 20px);
}

/* ИЛИ классический подход с медиа-запросами */
html {
font-size: 18px; /* Десктоп по умолчанию */
}

@media (max-width: 768px) {
html {
font-size: 16px; /* Мобильные */
/* ВСЁ САЙТА УМЕНЬШИТСЯ ПРОПОРЦИОНАЛЬНО! */
}
}

@media (min-width: 1920px) {
html {
font-size: 20px; /* 4K-экраны */
/* ВСЁ САЙТА УВЕЛИЧИТСЯ ПРОПОРЦИОНАЛЬНО! */
}
}

/* Компоненты автоматически адаптируются */
.container {
padding: 2rem; /* 36px на десктопе */
max-width: 80rem; /* 1440px на десктопе */
font-size: 1.125rem; /* 20.25px на десктопе */
}

🔥 Новое в 2026: CSS-функция clamp()

Одна строка вместо медиа-запросов:

:root { /* Минимум 16px, идеал 1vw + 14px, максимум 20px */ font-size: clamp(16px, 1vw + 14px, 20px); }

Это создаёт плавное масштабирование без резких скачков на брейкпоинтах!

🚫 Чего точно НЕ делать в 2026

❌ Запрещено

  • Использовать px для font-size (нарушает доступность)
  • Смешивать px и rem без системы в одном компоненте
  • Задавать line-height в px (используйте безразмерные значения: 1.5, 1.6)
  • Забывать про пользовательские настройки масштаба

✅ Обязательно

  • Определить :root { font-size: ... } в начале CSS
  • Использовать CSS-переменные для единообразия
  • Тестировать с увеличенным масштабом браузера (200%)
  • Согласовать подход в команде (единая методология)

🏆 Итог и рекомендация на 2026 год

🎯 Главный принцип 2026: "REM-first approach"

Начинайте с REM для всего, затем точечно используйте PX там, где нужна абсолютная точность.

/* РЕКОМЕНДУЕМАЯ СТРУКТУРА 2026 */
:root {
--base-font: clamp(16px, 1vw + 14px, 20px);
--spacing-unit: 0.5rem; /* 9px при базисе 18px */
font-size: var(--base-font);
}

.component {
/* Всё в REM */
font-size: 1.125rem;
padding: calc(var(--spacing-unit) * 3);
margin-bottom: 2rem;

/* Исключения в PX */
border: 1px solid #ddd;
box-shadow: 0 2px 8px rgba(0,0,0,.1);
}

Тренды января 2026:

  1. clamp() вместо медиа-запросов для базового размера
  2. CSS-переменные для spacing system (0.25rem, 0.5rem, 1rem...)
  3. Figma-плагины, которые экспортируют сразу в REM
  4. Автоматический конвертер в сборке (PostCSS плагины)
  5. Базовый размер 18-20px вместо старых 16px (лучшая читаемость)

Будущее (2027+): container units и dynamic viewport

Следите за появлением cqw, cqh (контейнерные единицы) и dvh, dvw (динамические единицы вьюпорта).
