Высоты: height vs min-height

height:

  • жёсткая высота
  • контент не может увеличить элемент
  • часто ломает layout
.modal {
   height: 100dvh;
   }

Использовать для:

  • модалок
  • fullscreen-меню
  • overlay

min-height:

  • минимальная высота
  • контент может увеличить элемент
.body {
    min-height: 100dvh;
    }

Использовать для:

  • body
  • layout страниц
  • hero-блоков
  • sticky footer

vh vs dvh:

  • 100vh — статичная высота viewport
  • 100dvh — динамическая, учитывает адресную строку на мобильных
.section {
    min-height: 100dvh;
    }
Рекомендуется для современных проектов

Flex: две колонки (аналог grid-template-columns: 1fr 160px)

.header {
    display: flex;
      padding: 30px 0 70px 20px;
    }

    .title {
    flex: 1; /* занимает всё доступное */
    }

    .quote {
    width: 160px; /* фиксированная колонка */
    flex-shrink: 0; /* запрещаем сжиматься */
   }

Важное правило Flex:

Flex-элемент не может быть шире контейнера.

Если что-то не растягивается — проверь:

  • ширину родителя
  • flex-shrink
  • overflow

CSS-переменные (var())

Объявление

:root {
    --border: 2px solid #000;
    --radius: 4px;
    }

Почему border-radius работает, а border — нет


  :border-radius: var(--radius); /* ✔ работает */
  border: var(--border); /* ❌ может не работать */

Причина:

  • border-radius — одно значение
  • border — shorthand (width + style + color)
  • если хоть часть невалидна → всё свойство отбрасывается

Надёжный вариант с fallback


   border: var(--border, 2px solid black);

Если переменная недоступна — используется запасное значение.

Диагностический тест переменных


   border: var(--border, 5px solid lime);

Результат:

  • 🟩 lime появился → переменная не доступна
  • ⬛ чёрный → переменная есть, но возможно переопределена ниже

Частые проблемы

  • :root в CSS Module без :global
  • CSS-файл не импортирован
  • переменная переопределена ниже
  • shorthand без fallback

Ментальная модель

CSS-переменная — это ссылка, а не значение.

Если ссылка не резолвится → браузер берёт fallback
или игнорирует свойство.

Базовые правила

  • Layout → min-height, не height
  • Body → min-height: 100dvh
  • Modals → height: 100dvh
  • Flex → flex: 1 + flex-shrink: 0
  • Shorthand → всегда с fallback
