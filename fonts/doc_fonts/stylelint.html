Полная инструкция по настройке и использованию Stylelint

Установка и базовая настройка

  1. Установка пакетов 
    
# Перейти в корень проекта и выполнить:
# Или базоваую установку
  npm install --save-dev stylelint stylelint-config-standard

# Или расширенную установку (рекомендуется)
  npm install --save-dev stylelint stylelint-config-standard stylelint-config-html stylelint-scss postcss-html postcss-scss
  2. Создание конфигурационного файла

    Создаем файл .stylelintrc.json в корне проекта:

    json
        {
  // Базовые конфиги для расширения
   "extends": [
    "stylelint-config-standard",  // Стандартные правила
    "stylelint-config-html"       // Поддержка HTML/Vue файлов
  ],
  
  // Плагины для дополнительного функционала
  "plugins": [
    "stylelint-scss"              // Поддержка SCSS/Sass
  ],
  
  // Кастомные правила
  "rules": {
    // Цвета в нижнем регистре: #fff вместо #FFF
    "color-hex-case": "lower",
    
    // Короткая запись цветов: #fff вместо #ffffff
    "color-hex-length": "short",
    
    // Отключение проверки имен классов (можно использовать любые имена)
    "selector-class-pattern": null,
    
    // Отключение проверки имен анимаций
    "keyframes-name-pattern": null,
    
    // Отключение проверки CSS переменных
    "custom-property-pattern": null,
    
    // Разрешение использования сокращенных свойств
    "declaration-block-no-redundant-longhand-properties": null,
    
    // Дополнительные полезные правила:
    "indentation": 2,                           // Отступы 2 пробела
    "string-quotes": "single",                  // Одинарные кавычки
    "max-nesting-depth": 3,                     // Макс. вложенность 3 уровня
    "no-descending-specificity": null,          // Отключение проверки специфичности
    "selector-pseudo-class-no-unknown": [true, { "ignorePseudoClasses": ["global"] }]
  },
  
  // Настройки для разных типов файлов
  "overrides": [
    {
      "files": ["**/*.html", "**/*.vue"],
      "customSyntax": "postcss-html"
    },
    {
      "files": ["**/*.scss"],
      "customSyntax": "postcss-scss"
    }
  ]
}
  3. Настройка VS Code

    Добавляем в настройки VS Code (Ctrl+, → "Open Settings (JSON)"):

    json
        {
  // Отключаем встроенную валидацию CSS/SCSS
  "css.validate": false,
  "less.validate": false,
  "scss.validate": false,
   
  // Автоматическое исправление при сохранении
  "editor.codeActionsOnSave": {
  "source.fixAll.stylelint": true
  },
  
  
  // Форматирование при сохранении
  "editor.formatOnSave": true,
  
  // Указываем Stylelint как форматтер для CSS/SCSS
  "[css]": {
    "editor.defaultFormatter": "stylelint.vscode-stylelint" 
  },
  "[scss]": {
    "editor.defaultFormatter": "stylelint.vscode-stylelint"
  }
}

Практическое использование с учебным конфигом

Ежедневно (быстрая работа):


        stylelint "**/*.css" --fix

Перед сдачей работы в учебную часть (проверка по стандартам):


        stylelint "**/*.css" --config .stylelintrc.academy.json

Способ увидеть изменения:


        # Показать все изменения в CSS файлах
git diff style/

# Или только конкретный файл
git diff style/style.css

# С подсветкой слов (как вы уже делали)
git diff --word-diff style/style.css

# Краткая статистика по всем файлам
git diff --stat
