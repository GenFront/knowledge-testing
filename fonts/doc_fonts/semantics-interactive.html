Кнопки / Ссылки — используем правильные элементы

❌ Частая ошибка

<div class="btn">Отправить</div>

✅ Правильно

<button type="button">Отправить</button>
Демонстрация
div как кнопка

Когда <div> — уместен (и даже нужен)

  • Для визуальной группировки внутри семантического блока (Flex / Grid контейнер).
  • Как вспомогательный layout-контейнер в компонентах.
  • Для анимаций, декоративных эффектов, обёрток без смысловой нагрузки.

Хороший пример

<section>
  <div class="cards-grid">
    <article>...</article>
    <article>...</article>
  </div>
</section>

Плохой пример

<div><div><div>...</div></div></div>

ARIA / role — как придать смысл <div>

Используй ARIA, только если семантический тег нельзя применить.

Без роли

<div onclick="play()">▶ Смотреть видео</div>

С ролью и доступностью

<div role="button" tabindex="0" aria-label="Смотреть видео" onclick="play()">▶ Смотреть видео</div>
Демонстрация (блок с ролью)
▶ Смотреть видео (div)

Нажми Enter/Пробел на элементе с ролью — событие обрабатывается (в демо ниже).

ARIA подсказки
role="button" — делает элемент Button для скринридеров
tabindex="0" — делает элемент доступным по Tab
aria-label="..." — имя для ассистивных технологий
